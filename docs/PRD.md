# PalmSugarReader 产品需求文档（PRD）

> 版本：v0.1 MVP | 更新：2026-06-03

---

## 1. 产品定位

PalmSugarReader 是一款**轻量化、跨平台**的综合阅读浏览器。不做臃肿的文档编辑器，专注**阅读体验 + 标注能力**，填补"极光 PDF 太重、Adobe 太贵、福昕不更新"之间的空白。

长期目标：融入 AI 学习增强功能，服务于教学场景（日语、EJU 等）。

## 2. 目标用户

- 日常阅读者：需要一款轻量多格式阅读器
- 学生/自学者：需要标注、翻译、知识点提取
- 教学者：需要内容分发与阅读体验可控的平台

## 3. 技术栈

| 决策 | 选型 | 理由 |
|------|------|------|
| 跨平台框架 | Flutter | 三端统一代码，MVP 快速验证 |
| 桌面备选 | Tauri + Flutter | 若 Windows 端 Flutter 打包体积过大时启用 |
| 状态管理 | 待定 | Provider / Riverpod / Bloc，MVP 前确定 |

## 4. 第一期功能（MVP）

### 4.1 多格式阅读

| 优先级 | 格式 | 说明 |
|--------|------|------|
| P0 | PDF | 核心格式，需支持文字选择和缩放 |
| P0 | EPUB | 主流电子书格式，需支持回流排版 |
| P0 | TXT | 纯文本，自动检测编码 |
| P0 | Markdown | 核心中转格式，需渲染为富文本 |
| P0 | 图片 | JPEG/PNG/WebP 等常见图片格式查看 |
| P1 | MOBI | Kindle 旧格式 |
| P1 | DOCX | Word 文档阅读 |
| P2 | AZW/AZW3 | Amazon DRM 格式，解析难度高 |
| P2 | DOC | 旧版 Word 格式 |

### 4.2 书签

- 记录阅读位置（文件 + 页码/进度）
- 书签列表管理（增删改查）
- 支持跨会话持久化

### 4.3 标注功能

- **高亮**：选中文字后高亮标记
- **划线**：下划线标注
- **批注**：附加文字笔记
- **设计原则**：不修改原文内容，保护版权；重度编辑请用 Typora / Word

### 4.4 格式转换

- 以 Markdown 为核心中转格式
- 支持格式互转：PDF → MD、EPUB → MD、MD → PDF 等
- 优先支持：TXT ↔ MD、EPUB → MD、MD → PDF

### 4.5 三端覆盖

- Windows 桌面端
- Android 移动端
- iOS 移动端

## 5. 非功能需求

- **轻量化**：安装包尽可能小，内存占用合理
- **UI 友好**：界面简洁、操作直观、降低学习成本
- **多语言**：后期开放 i18n（中日英优先）

## 6. 未来规划（第二期及以后）

### 6.1 用户系统
- 注册 / 登录 / 个人中心
- 阅读进度云同步
- 标注数据云备份

### 6.2 内容商城
- 教学资料、电子书数字内容分发
- 与用户系统打通

### 6.3 AI 学习增强
- **自学辅助**：阅读理解、知识点提取、笔记整理、思维导图生成
- **翻译功能**：文档段落翻译，保持排版；结合字幕组经验做高质量翻译
- **智能标注**：AI 自动识别重点内容并标注，自动生成摘要

### 6.4 多语言版本
- 界面国际化（中日英优先）

## 7. 竞品参考

| 产品 | 优点 | 缺点 |
|------|------|------|
| 极光 PDF | 功能全面 | 臃肿、收费 |
| Adobe Acrobat | 行业标准 | 太重、贵 |
| 福昕 PDF | 老牌 | 更新慢 |
| Kindle | 生态好 | 格式封闭 |
| Calibre | 格式转换强 | UI 老旧 |

## 8. 设计决策记录

1. **编辑 ≠ 改原文**：只做标注层，不改文档内容，保护版权 + 保持轻量
2. **MD 是核心中转格式**：格式转换以 Markdown 为枢纽
3. **Flutter 先行**：先用一套代码验证，后期按需拆分
4. **标注数据独立存储**：与原文分离，方便同步和迁移
5. **主色调 #A3C1AD**：鼠尾草绿（Sage Green），用于品牌标识、主题色、强调色等 UI 元素，传递轻量、自然、友好的产品气质
