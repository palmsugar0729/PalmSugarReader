# PalmSugarReader 产品需求文档（PRD）

> 版本：v0.2 MVP | 更新：2026-06-13

---

## 1. 产品定位

PalmSugarReader 是一款**轻量化、跨平台**的综合阅读浏览器。不做臃肿的文档编辑器，专注**阅读体验 + 标注能力**，填补"极光 PDF 太重、Adobe 太贵、福昕不更新"之间的空白。

长期目标：融入 AI 学习增强功能，服务于教学场景（日语、EJU 等）。

## 2. 目标用户

- 日常阅读者：需要一款轻量多格式阅读器
- 学生/自学者：需要标注、翻译、知识点提取
- 教学者：需要内容分发与阅读体验可控的平台

## 3. 技术栈

| 决策 | 选型 | 理由 |
|------|------|------|
| 跨平台框架 | Flutter | 三端统一代码，MVP 快速验证 |
| 桌面备选 | Tauri + Flutter | 若 Windows 端 Flutter 打包体积过大时启用 |
| 状态管理 | 待定 | Provider / Riverpod / Bloc，MVP 前确定 |

## 4. 第一期功能（MVP）

### 4.1 多格式阅读

| 优先级 | 格式 | 说明 |
|--------|------|------|
| P0 | PDF | 核心格式，需支持文字选择和缩放 |
| P0 | EPUB | 主流电子书格式，需支持回流排版 |
| P0 | TXT | 纯文本，自动检测编码 |
| P0 | Markdown | 核心中转格式，需渲染为富文本 |
| P0 | 图片 | JPEG/PNG/WebP 等常见图片格式查看 |
| P1 | MOBI | Kindle 旧格式 |
| P1 | DOCX | Word 文档阅读 |
| P2 | AZW/AZW3 | Amazon DRM 格式，解析难度高 |
| P2 | DOC | 旧版 Word 格式 |

### 4.2 书签

- 记录阅读位置（文件 + 页码/进度）
- 书签列表管理（增删改查）
- 支持跨会话持久化

### 4.3 标注功能

- **高亮**：选中文字后高亮标记
- **划线**：下划线标注
- **批注**：附加文字笔记
- **设计原则**：不修改原文内容，保护版权；重度编辑请用 Typora / Word

### 4.4 格式转换 ✅（已实现）

- 以 Markdown 为核心中转格式
- 直接转换：TXT → MD、MD → TXT、EPUB → MD、MD → PDF
- 链式转换：通过 MD 中转支持更多组合（如 EPUB → MD → PDF）
- TXT 转换自动多编码检测，MD 前端含 YAML 元数据
- MD → PDF 使用 Noto Sans SC 字体渲染中日文
- 转换入口：阅读器 AppBar 菜单 → 选择目标格式 → 保存 → 结果提示

### 4.5 阅读体验增强（2026-06-13 用户反馈）

#### 4.5.1 缩放控制增强
- **现状**：PDF 右下角有 +/- 按钮；图片阅读器支持手势缩放
- **待做**：
  1. Ctrl + 滚轮缩放（桌面端标准交互）
  2. 缩放百分比滑条（位于 +/- 按钮附近，拖动精确调节）
  3. 当前缩放比例数值显示
  4. 双击/右键重置为 100%

#### 4.5.2 键盘快捷键
- **PgUp / PgDn**：上下翻页
- **Home / End**：跳到文档开头/末尾
- **Ctrl + 滚轮**：缩放（见 4.5.1）
- **F2**：重命名（配合右键菜单）
- **Ctrl+C / Ctrl+X / Ctrl+V**：复制/剪切/粘贴

#### 4.5.3 右键上下文菜单
- **现状**：首页支持左滑删除（已优化阈值 25%）、长按多选
- **待做**：
  1. 首页文件列表项右键弹出菜单（类 Windows 资源管理器）
  2. 菜单内容待讨论：删除、重命名、格式转换快捷入口、文件信息等
  3. 阅读器内右键菜单：复制、选择全部、缩放重置等

#### 4.5.4 会话记忆 / 最近文件持久化
- **现状**：`_recentBooks` 仅内存保存，重启即清空
- **待做**：
  1. 最近打开的文件列表持久化到本地（`shared_preferences` 或 JSON 文件）
  2. 跨会话保留阅读进度（最后阅读位置/页码）
  3. 支持清除历史
- **方案选择**：轻量用 JSON 文件（`path_provider` + `dart:io`），后续上 `shared_preferences` 或 SQLite

#### 4.5.5 设置页面
- **待做**：
  1. 入口：首页右上角齿轮图标（已实现占位）
  2. 设置项初稿：
     - 批量导入开关（见 4.5.6）
     - 默认编码（自动检测 / 手动指定）
     - 阅读器默认缩放比例
     - 主题色（预设 / 自定义）
     - 语言（中日英，预留）
     - 清除历史 / 重置设置

#### 4.5.6 批量导入（相似文件自动发现）
- **需求**：添加文档后，自动扫描当前文件所在目录，识别并导入所有扩展名相同的文件
- **逻辑**：
  1. 用户打开（或拖入）`/docs/chapter3.txt`
  2. 后台扫描 `chapter*.txt` → `chapter1.txt`, `chapter2.txt`...
  3. 弹出提示："发现 5 个相似文件，是否批量导入？"
- **开关**：默认关闭，在设置中开启
- **范围限制**：仅匹配同目录、同扩展名，含数字/页码模式的相似文件名

#### 4.5.7 EPUB 大文件性能修复 ✅（已修复）
- **问题**：大 EPUB 加载卡顿、跳页
- **根因**：Controller 创建时 `Future<EpubBook>` 未完成就隐藏了 loading
- **修复**：`await EpubReader.readBook()` 同步等待解析完成后再创建 Controller

## 8. 设计决策记录

1. **编辑 ≠ 改原文**：只做标注层，不改文档内容，保护版权 + 保持轻量
2. **MD 是核心中转格式**：格式转换以 Markdown 为枢纽
3. **Flutter 先行**：先用一套代码验证，后期按需拆分
4. **标注数据独立存储**：与原文分离，方便同步和迁移
5. **主色调 #A3C1AD**：鼠尾草绿（Sage Green），用于品牌标识、主题色、强调色等 UI 元素，传递轻量、自然、友好的产品气质
6. **PDF 字体**：嵌入 Noto Sans SC（思源黑体，SIL 开源），17MB，覆盖中/日/英。Release 包从 45MB → 62MB
7. **EPUB 加载策略**：大文件同步 `await EpubReader.readBook()`，确保解析完成再渲染，避免跳页
8. **滑动删除**：阈值从默认 50% 降至 25%，动画 200ms，轻扫即触发
9. **文件持久化**：近期文件记忆用 JSON 文件（`path_provider` + `dart:io`），不引入额外依赖
