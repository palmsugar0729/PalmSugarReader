# PalmSugarReader 产品需求文档（PRD）

> 版本：v0.6 MVP | 更新：2026-06-22

---

## 1. 产品定位

PalmSugarReader 是一款**轻量化、跨平台**的综合阅读浏览器。不做臃肿的文档编辑器，专注**阅读体验 + 标注能力**，填补"极光 PDF 太重、Adobe 太贵、福昕不更新"之间的空白。

长期目标：融入 AI 学习增强功能，服务于教学场景（日语、EJU 等）。

## 2. 目标用户

- 日常阅读者：需要一款轻量多格式阅读器
- 学生/自学者：需要标注、翻译、知识点提取
- 教学者：需要内容分发与阅读体验可控的平台

## 3. 技术栈

| 决策 | 选型 | 理由 |
|------|------|------|
| 跨平台框架 | Flutter | 三端统一代码，MVP 快速验证 |
| 桌面备选 | Tauri + Flutter | 若 Windows 端 Flutter 打包体积过大时启用 |
| 状态管理 | 待定 | Provider / Riverpod / Bloc，MVP 前确定 |

## 4. 第一期功能（MVP）

### 4.1 多格式阅读

| 优先级 | 格式 | 说明 |
|--------|------|------|
| P0 | PDF | 核心格式，文字选择 + 缩放（Ctrl+滚轮/滑条） |
| P0 | EPUB | 自研翻页阅读器，懒加载秒开，水平翻页 |
| P0 | TXT | 纯文本，多编码自动检测 |
| P0 | Markdown | 核心中转格式，LaTeX 公式渲染 |
| P0 | 图片 | JPEG/PNG/WebP/GIF/BMP 查看 |
| P2 | DOCX | Word 文档阅读（以后加） |
| P2 | DOC | 旧版 Word 格式（以后加） |

### 4.2 书签 ✅（已实现）

- 记录阅读位置（文件 + 页码 + 章节信息）
- 书签列表底部弹窗管理（增删 + 点击跳转 + 滑动删除）
- 支持跨会话持久化（per-file JSON，与标注同模式）
- 仅 PDF / EPUB 支持（`book.isBookmarkable`）

### 4.3 标注功能 ✅（已实现）

- **统一标注图层**：`AnnotationLayer` 组件，透明画布盖在内容层上，相对坐标归一化，缩放跟随
- **高亮**：鼠标自由拖拽画矩形，固定高度(=粗细)，颜色+不透明度可调
- **划线**：同高亮，渲染为底部细线
- **批注（浮动便签）**：点击页面放置可拖动便签，点击查看/编辑文字，支持删除
- **自由画笔**：手写路径，支持铅笔/画笔/水彩笔三种笔刷，贝塞尔平滑
- **橡皮擦**：白色粗笔模拟擦除，桌面端 Dialog + 平板工具栏均可使用
- **撤销**：支持撤销最后一笔标注（`AnnotationService.popLast`）
- **颜色选择器**：预置常用色（黄/绿/蓝/粉/橙/黑）+ HEX 输入 + 不透明度滑条 + 粗细滑条（2~40px）
- **平板端浮动工具栏**（2026-06-22）：GoodNotes 风格常驻底部工具栏 — 即时切换工具/颜色/粗细，可折叠，无需退出标注模式
- **防误触**（2026-06-22）：仅响应触控笔（`PointerDeviceKind.stylus`），手掌/手指自动忽略；可选开启"手指书写"
- **付费预留**：`AnnotationService.enabled` 开关
- **设计原则**：不修改原文内容，保护版权
- **Bug 修复 ✅**：2026-06-22 修复退出标注模式后已保存标注不可见（渲染层与交互层拆分）

### 4.3.1 字体策略 ✅（已实现）

采用「系统字体优先 + 按需加载」方案：

1. PDF：`pdfrx` 自动使用文档内嵌字体
2. EPUB/MD/TXT：Flutter 默认调用系统 CJK 字体（Windows: Microsoft YaHei / SimHei / SimSun，iOS/macOS: PingFang，Android: Noto Sans CJK）
3. MD→PDF 转换：完整字体优先级链（用户导入 → 系统字体 → 内置 NotoSansSC-Regular）
4. 设置页支持用户手动导入额外字体作为 fallback（已用于 MD→PDF 转换）
5. 内置 Noto Sans SC Regular（~10MB）已替换原 VF（17MB），作为最后兜底

> 字体链仅渗透到 MD→PDF 转换器，屏幕阅读器依赖系统字体。对中日文目标用户群，系统字体已覆盖所有场景，MVP 阶段不再深入。

### 4.4 格式转换

#### 已实现 ✅
- TXT ↔ MD（自动编码检测 + YAML 元数据）
- EPUB → MD（保留 epub_md_converter 供链式中转）
- MD → PDF（Noto Sans SC 字体渲染中日文）
- MD → EPUB ✅（2026-06-19 EPUB 3 标准结构：nav.xhtml + CSS + 封面 + 章节自动分割）
- MD → TXT ✅（2026-06-19 Markdown AST 提取纯文本）
- 图片 → PDF ✅（2026-06-19 单图/多图，一图一页 / 紧凑 2×2 网格）

#### 待做（2.0）
| 源 → 目标 | 说明 |
|-----------|------|
| MD → DOCX | 以后加 |
| TXT → PDF | 通过 MD 中转（链式转换已支持）|

#### 不做
- EPUB → MD 入口删除（仅保留内部中转能力）
- TXT → EPUB（无章节结构，意义不大）
- MOBI 输出（格式太老，Kindle 已转向 KF8）
- AZW/AZW3

## 8. 设计决策记录

1. **编辑 ≠ 改原文**：只做标注层，不改文档内容，保护版权 + 保持轻量
2. **MD 是核心中转格式**：格式转换以 Markdown 为枢纽
3. **Flutter 先行**：先用一套代码验证，后期按需拆分
4. **标注数据独立存储**：与原文分离，方便同步和迁移
5. **主色调 #A3C1AD**：鼠尾草绿（Sage Green），传递轻量、自然、友好的产品气质
6. **字体策略**：系统字体优先，内置精简单字重兜底（~5MB），用户可手动导入 fallback
7. **EPUB 翻页**：放弃 epub_view，自研 epubx 懒加载 + PageView 方案，大文件秒开
8. **滑动删除**：阈值 25%，动画 200ms，轻扫即触发
9. **文件持久化**：JSON 文件（`path_provider` + `dart:io`），不引入额外依赖
10. **用户系统**：腾讯云自建后端，测试期本地模拟
11. **标注坐标归一化**：全部存 0~1 相对比例，天然缩放跟随，不依赖像素绝对坐标
12. **便签批注**：浮动贴纸式，可拖动移动位置，点击查看编辑
13. **产品定位 PotPlayer 类比**：轻量启动、格式通吃、功能深藏不臃肿 — 默认界面极简，需要时一个手势/按钮唤出功能面板；MVP 优先，看反馈再迭代
14. **平板标注工具栏 GoodNotes 模式**：常驻底部浮动工具栏，即时切换工具/颜色/粗细，可折叠；桌面端保留 Dialog 流程
15. **防误触**：基于 `PointerDeviceKind.stylus` 过滤，默认仅响应触控笔
