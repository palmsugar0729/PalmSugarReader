# 批量导入文件重启后消失

**状态**：🟢 已解决
**发现日期**：2026-06-19
**关联**：[[2026-06-19_开发日志]]

## 现象

用户通过"批量导入"一次性导入多个同目录文件，当时列表显示正常，但关闭应用重启后，这些导入的文件没有出现在最近列表中。

## 根因

`_checkSiblingFiles` 中的保存逻辑：

```dart
for (final siblingPath in siblings) {
  final siblingBook = Book.fromFile(siblingPath);
  if (siblingBook.isReadable) {
    _recentBooks.insert(0, siblingBook);
    BookmarkService.addOrUpdate(siblingBook); // ⚠️ 没有 await！
    imported++;
  }
}
```

`BookmarkService.addOrUpdate` 是异步的（读写 JSON 文件），但循环中没有 `await`。当导入文件较多时，所有保存操作还在队列中，用户可能已经关闭了应用，导致文件未被持久化。

## 解决方案

使用 `Future.wait` 等待所有保存完成：

```dart
final saveFutures = <Future<void>>[];
for (final siblingPath in siblings) {
  final siblingBook = Book.fromFile(siblingPath);
  if (siblingBook.isReadable) {
    _recentBooks.insert(0, siblingBook);
    saveFutures.add(BookmarkService.addOrUpdate(siblingBook));
    imported++;
  }
}
await Future.wait(saveFutures);
```

同时把 `_addAndOpenBook` 也改为 `async` + `await`：

```dart
Future<void> _addAndOpenBook(Book book) async {
  _recentBooks.insert(0, book);
  await BookmarkService.addOrUpdate(book); // 确保保存完成再打开
  _openBook(book);
}
```

## 预防

- [ ] 任何涉及文件/数据库持久化的异步操作，在循环中必须使用 `await` 或 `Future.wait`
- [ ] review 所有 `Service.xxx()` 调用，确认是否遗漏 `await`
